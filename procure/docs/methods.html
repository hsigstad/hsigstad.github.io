Empirical Methods

Methods

Court-to-procurement linking

Identification strategy

  • Extract procurement process references from TJSP court decision text (sentenças)
  • Join extracted references to TCE-SP ajustes (contracts) on (municipality, proc_num, proc_year)
  • Use Código da Licitação to reach bid-level data in licitação files

Pipeline

  • script: source/linking/match_court_to_procurement.py
  • inputs: TJSP sentenças (12,433 improbidade cases), TCE-SP ajustes (3,991,407 rows, 2016–2022)
  • outputs: build/linking/court_to_procurement.csv, build/linking/court_refs_unmatched.csv

Extraction patterns

  • three regex patterns applied in order:
    1. modality keyword + [nº] + num/year (high confidence)
    2. nº X/YYYY with procurement context within 300 characters (medium confidence)
    3. PA (processo administrativo) prefix with procurement context
  • list continuation: captures comma-separated items after initial modality match
  • exclusions: legal references (Lei, Decreto, etc.), known law numbers, years outside 1990–2025

Municipality normalization

  • comarca (court jurisdiction) mapped to ajustes Município
  • normalization: strip accents, lowercase, remove "/ SP" suffix, hyphens→spaces
  • 307/312 comarcas resolved (98.4%)

Key assumptions

  • comarca name corresponds to the municipality where the procurement occurred
  • procurement process numbers in court text refer to the same numbering system as TCE-SP ajustes
  • 2-digit years: ≤30 → 2000s, >30 → 1900s

Robustness

  • extraction quality evaluated against GPT-4o-mini on two independent random samples (n=50 each)
  • regex recall: ~96–100% against true procurement references
  • most apparent FNs were LLM errors (hallucinated numbers, non-procurement references)
  • precision is less critical: false positive extractions fail to join and drop out
  • see docs/briefs/linking.md for full evaluation details
